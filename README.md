4.学习率分析
考核要求我们尝试不同的学习率，我做了三组实验：

实验1：学习率=1.0（太大）
- 现象：Loss一直震荡在162.7左右，降不下去
- 原因：学习率太大，参数更新步子迈太大，直接跳过了最优点

实验2：学习率=0.01（正常）
- 现象：Loss平稳下降，最终到161.88
- 原因：学习率适中，稳步收敛

实验3：学习率=0.00001（太小）
- 现象：Loss下降非常慢，1000轮只降到191.95
- 原因：学习率太小，参数更新步子太小，走不动

5. 过拟合判定
考核还要求我们观察过拟合现象，我做了两组对比：

正常模型（未过拟合）
- 网络：3层 (1-10-1)
- 轮数：1000
- 现象：Loss平稳下降，拟合曲线光滑，符合数据趋势

过拟合模型
- 网络：加深到5层 (1-100-100-100-1)
- 轮数：10000
- 现象：
  - Loss降到了0.5以下（几乎为0）
  - 拟合曲线剧烈扭曲，试图穿过每一个数据点
- 判断依据：训练Loss极低，但拟合曲线不光滑，说明模型把噪声也学进去了

任务四
1. 模型结构改进
原来的模型（任务三）：
· 3层MLP：3072 → 512 → 256 → 10
· 参数量：1,707,274

改进后的模型（任务四）：
· CNN + 残差连接
· 结构：卷积层(3→32) → 残差块×2(32) → 残差块×2(64) → 残差块×2(128) → 全局池化 → 全连接(10)
· 参数量：696,650（比原来少了100万）

为什么这么改：

· CNN能提取图片的空间特征（边缘、纹理），MLP只能看像素点
· 残差连接让梯度更好传，网络可以 deeper

2. 训练策略改进
加了Warmup（学习率预热）：
· 前5轮：学习率从0慢慢升到0.001
· 后15轮：余弦下降到0
· 目的：刚开始模型是乱的，小步走稳一点

加了Mixup（数据混合）：
· 公式：新图片 = λ×图A + (1-λ)×图B
· 新标签也要混合算损失
· 目的：让模型学得更泛化，不要死记硬背

二、实验结果
跑了5组对比：
实验设置 准确率
MLP（原始） 53.0%
CNN（不加Tricks） 89.0%
CNN + Mixup 87.7%
CNN + Warmup 89.6%
CNN + Warmup + Mixup 88.1%
结果解读：
1. MLP→CNN：提升了36%，效果最明显
2. 加Warmup：再提0.6%，有点用
3. 加Mixup：反而降了1.3%，没起作用
4. 两个一起加：88.1%，不如单独用Warmup


三、失败复盘
Mixup为什么没起作用？
分析的原因：
1. 数据集问题：CIFAR-10图片只有32×32像素，混合后更模糊，模型看不清
2. 参数问题：α设成1.0可能太大，混合得太狠了
3. 轮数问题：只跑了20轮，Mixup可能需要更多轮数才能见效
下次改进：
· 把α调小到0.2试试
· 跑50轮看看

八、分割vs分类的数据处理差异
1 主要区别
方面 分类任务 分割任务
标签形式 单个数字（0,1,2...） 和原图一样大的矩阵
信息量 图片属于哪类 每个像素属于哪个实例
数据加载 直接读取标签 需要解析Mask文件
数据增强 只增强图片 图片和Mask必须同步增强
2 遇到的难点及解决
 难点1：Mask解析
   · 问题：Mask文件中不同数字代表不同实例，需要分离
   · 解决：用 np.unique(mask) 获取实例ID，再逐个生成二值Mask
 难点2：边界框计算
   · 问题：需要从Mask中提取每个实例的Bounding Box
   · 解决：用 np.where(mask) 找到非零像素，取最小/最大坐标
 难点3：数据加载格式
   · 问题：Mask R-CNN需要特定的target格式（boxes, labels, masks, area等）
   · 解决：参考PyTorch官方文档，构建包含所有必要字段的字典
